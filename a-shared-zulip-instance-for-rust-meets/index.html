a shared Zulip instance for Rust meets

2025-07-09

Would be kool to have a shared place for people of various Rust meets around the world.

An idea I have is that each meet group would have its own channel, and then there would be shared channels, where people would, for example, share tips on making their meets more inclusive, or whatever else.

A possible structure:

- misc
  - news (for sharing some Rust links)
  - off topic
- tips
  - food
  - attracting talks
  - ...
- group/johannesburg (a location where meet happens)
  - 2025-07-08 (notes on an individual meet)
- group/berlin
  ...
- group/nairobi
  ...

I have created one, https://rust-meets.zulipchat.com. Let's see how far that goes.

Why Zulip:

  • It is used by Rust makers (instance), as well as some adjacent communities, like Rust for Linux (instance) and Servo (instance)
  • It is open source
  • It has amazing usability (or at least the best software of its type I used yet)
  • It is developed in public (see https://chat.zulip.org)
